## Summary

Move 3 interfaces defined locally in service files to the `port/` package where they belong per clean architecture. Create port interfaces for services used by REST handlers so adapters depend on abstractions, not concrete service types.

## Context

The style guide mandates that interfaces live in `port/` and adapters depend only on port interfaces. Currently, `service/project_service.go` defines `ProjectGiteaClient` and `ProjectStoreWriter` locally, `service/board_service.go` defines `NativeBoardStore` locally, and `adapter/rest/api_handler.go` imports concrete `service.*` types directly.
