## Summary

Add UI controls to the dashboard for registering and removing projects. An "Add Project" form lets users provide a remote URL, and a "Remove" action on each project row handles deregistration with a confirmation dialog.

## Context

The dashboard's Overview page already lists registered projects in a table (slug, origin remote, track counts). Currently the only way to add a project is `kf add <remote>` from the terminal. Users should be able to manage projects directly from the dashboard.

### Current Dashboard State

- **Overview page** (`OverviewPage.tsx`) — lists projects, links to project detail
- **Project page** (`ProjectPage.tsx`) — shows metadata, kanban board, tracks
- **Data hook** (`useProjects.ts`) — fetches `GET /-/api/projects` (read-only)
- **Empty state** — shows "No projects registered — run `kf add <remote>`"
