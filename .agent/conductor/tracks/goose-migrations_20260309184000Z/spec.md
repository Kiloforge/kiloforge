## Technical Notes

### Migration file format

```sql
-- +goose Up
CREATE TABLE config (
    key   TEXT PRIMARY KEY,
    value TEXT NOT NULL
);
-- ... rest of V1 schema ...

-- +goose Down
DROP TABLE IF EXISTS spans;
DROP TABLE IF EXISTS traces;
-- ... drop in reverse order ...
```

### Embedding migrations

```go
import "embed"

//go:embed migrations/*.sql
var migrations embed.FS
```

Then in `Open()`:
```go
goose.SetBaseFS(migrations)
goose.SetDialect("sqlite3")
if err := goose.Up(db, "migrations"); err != nil {
    return nil, fmt.Errorf("migrate: %w", err)
}
```

### Handling existing schema_version table

Existing databases already have the V1 schema applied via the custom system. Options:
1. **Seed goose version** — On first goose run, if `schema_version` table exists with version >= 1, insert a corresponding row into `goose_db_version` so goose knows V1 is already applied. Then drop `schema_version`.
2. This ensures existing installs don't re-run the initial migration.
