## Summary

Add a global error notification system (toast) to the dashboard so that API errors, mutation failures, and other problems are visible to the user instead of silently failing or only appearing in the browser console.

## Context

Currently, when an API call fails (like the tour 400 error), the user sees nothing — the UI just doesn't respond. Errors are only visible in the browser developer console. Users need visible feedback when something goes wrong.

## Codebase Analysis

### Current error handling

- **TanStack Query mutations** — `onError` callbacks exist on some mutations but most just log to console or do nothing
- **TanStack Query queries** — `error` state is available but components inconsistently display it
- **No global error handler** — no centralized place to catch and display errors
- **Consent dialog** — the only existing "notification" pattern is the consent modal, which is task-specific

### Approach

Add a lightweight toast notification system:
1. **Toast context/provider** wrapping the app
2. **Global TanStack Query error handler** via `QueryClient` `onError` callback — catches all query/mutation errors automatically
3. **Manual toast API** for non-query errors (e.g., WebSocket disconnects, SSE failures)
4. **Auto-dismiss** after 5 seconds, with manual dismiss button

### Libraries

Could use a lightweight library like `react-hot-toast` or `sonner`, or build a simple custom toast since the app already uses Tailwind-style CSS. Keep it minimal.

## Acceptance Criteria

- [ ] Global toast notification component rendered at app root

## Technical Notes

### QueryClient global error handler

```tsx
const queryClient = new QueryClient({
  defaultOptions: {
    mutations: {
      onError: (error) => {
        toast.error(formatError(error));
      },
    },
  },
});
```

### Toast component

Position: top-right, stacked. Each toast shows:
- Error icon + message
- HTTP status if available
- Timestamp
- Dismiss button
