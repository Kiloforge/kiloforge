# Specification: Graceful Agent Shutdown and Auto-Recovery on Restart

## Summary

Implement graceful shutdown of all active CC agents (preserving session IDs), auto-resume on relay restart, and user-facing reporting for restore success/failure with clear error messages for unrestorable sessions.

## Context

Crelay spawns CC agents as child processes. When the relay shuts down, agents are killed without ceremony. Session IDs are stored in `state.json` but nothing uses them for auto-recovery. CC supports `claude --resume <sessionID>` for session restoration — already used in `ResumeDeveloper()` during review cycles.

## Codebase Analysis

- **Existing resume**: `defaultSpawner.ResumeDeveloper()` in relay server — `claude --resume <sessionID>`
- **Existing halt**: `Store.HaltAgent()` sends SIGINT
- **Session persistence**: `AgentInfo.SessionID` already in `state.json`
- **Shutdown signals**: `cli/up.go` and `cli/init.go` use `signal.NotifyContext` for SIGINT but no agent cleanup
- **`crelay stop`**: Stops individual agent, prints resume command — no batch stop
- **`crelay down`**: Stops Docker compose only, no agent shutdown

## Acceptance Criteria

- [ ] On relay shutdown (SIGINT/`crelay down`): all running agents receive SIGINT with configurable timeout
- [ ] Agents that were `running` at shutdown get status `suspended` (not `stopped` or `failed`)
- [ ] On relay startup (`crelay up`): detect `suspended` agents and auto-resume them
- [ ] Resume uses existing `claude --resume <sessionID>` mechanism

- [ ] Shutdown timeout configurable (default: 10 seconds), force-kill after timeout
- [ ] Worktree validation before resume: check worktree still exists and branch is intact

## Technical Notes

### Shutdown sequence

```
relay receives SIGINT
  → set shutting_down flag (reject new spawn requests)
  → for each running agent: send SIGINT, set status "suspending"
  → wait up to shutdown_timeout for all agents to exit
  → clean exits → status "suspended"
  → still running after timeout → SIGKILL → status "force-killed"
  → save state.json
  → stop relay server / Docker compose
```

### Resume-on-startup sequence

```
relay starts up
  → load state.json
  → find agents with status "suspended"
  → for each: validate worktree, attempt claude --resume <sessionID>
  → success → status "running", update PID
  → failure → status "resume-failed", store error
  → report results to user
```

### New AgentInfo fields

```go
ShutdownAt     *time.Time `json:"shutdown_at,omitempty"`
ShutdownReason string     `json:"shutdown_reason,omitempty"`
ResumeError    string     `json:"resume_error,omitempty"`
```

### New status values

- `suspended` — cleanly shut down, resumable
- `suspending` — SIGINT sent, waiting for exit
- `force-killed` — didn't exit within timeout
- `resume-failed` — resume attempted but failed
