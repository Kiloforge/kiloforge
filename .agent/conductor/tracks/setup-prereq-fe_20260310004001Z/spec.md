## Summary

Add frontend handling for the 428 "kiloforge setup required" response. When a user tries to generate tracks or spawn an agent on a project that hasn't been set up, show a dialog explaining that setup is needed and offer a button to launch an interactive setup session. After setup completes, the original operation is retried automatically.

## Context

The backend (track `setup-prereq-be_20260310004000Z`) returns HTTP 428 when conductor artifacts are missing for a project. The frontend needs to catch this error and guide the user through setup, following the same pattern used for consent (403 → ConsentDialog) and skills (412 → SkillsInstallDialog).

## Technical Notes

### useSetupPrompt hook

```typescript
export function useSetupPrompt() {
  const [isOpen, setIsOpen] = useState(false);
  const [projectSlug, setProjectSlug] = useState("");
  const retryRef = useRef<(() => void) | null>(null);

  const requestSetup = (slug: string, retryFn: () => void) => {
    setProjectSlug(slug);
    retryRef.current = retryFn;
    setIsOpen(true);
  };

  const handleSetupComplete = () => {
    setIsOpen(false);
    retryRef.current?.();
  };

  return { isOpen, projectSlug, requestSetup, handleSetupComplete, dismiss: () => setIsOpen(false) };
}
```

### SetupRequiredDialog

Similar to ConsentDialog/SkillsInstallDialog. The "Run Setup" button:
1. Calls `POST /api/projects/{slug}/setup`
2. Gets back agent info with WebSocket session ID
3. Opens the interactive terminal (same as existing interactive agent flow)
4. On agent completion, calls `handleSetupComplete` which retries the original operation

### Setup status indicator

The project page can poll `GET /api/projects/{slug}/setup-status` to show a banner like "Kiloforge Setup Required — Run setup to configure this project for track management" with a prominent action button.
