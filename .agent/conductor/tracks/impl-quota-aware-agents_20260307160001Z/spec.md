# Specification: Quota-Aware Agent Management and Cost Reporting

**Track ID:** impl-quota-aware-agents_20260307160001Z
**Type:** Feature
**Created:** 2026-03-07T16:00:01Z
**Status:** Draft

## Summary

Use the centralized quota tracker to implement spawn throttling when approaching rate limits, graceful backoff on 429 errors, cost-per-track reporting in `crelay status`, and user notifications when quota is running low.

## Context

With the quota tracker providing real-time token/cost metrics and rate limit detection, we can make the agent management layer quota-aware. This means not spawning new agents when rate limited, pausing/queuing spawn requests when approaching limits, showing users what each track is costing, and warning users before they hit hard quota limits.

## Codebase Analysis

- **Spawn entry points**: `Spawner.SpawnDeveloper()`, `Spawner.SpawnReviewer()`
- **CLI implement**: `internal/cli/implement.go` — calls spawner
- **Status command**: `internal/cli/status.go` — currently shows Gitea status only
- **Relay server**: `internal/relay/server.go` — spawns reviewers on webhook events
- **Tracker API** (from impl-quota-tracker): `IsRateLimited()`, `RetryAfter()`, `GetAgentUsage()`, `GetTotalUsage()`
