## Summary

The `kf up` path (`serve.go`) never calls `rest.WithInteractiveSpawner()`, so interactive agent endpoints return 500 "interactive agents not configured" when running via `kf up`. The previous fix (`fix-dashboard-interactive-agent`) only fixed the standalone `kf dashboard` command.

## Context

`serve.go` builds server options (lines 97-119) but never adds `WithInteractiveSpawner`. The `agent.Spawner` is never created in `serve.go` at all. Since `kf up` is the primary way to run Kiloforge, this means interactive agents, track generation from the dashboard, and admin operations are all broken in normal operation.

## Codebase Analysis

### Root Cause

**`backend/internal/adapter/cli/serve.go:97-119`** — builds `opts []rest.ServerOption` but never includes:
```go
spawner := agent.NewSpawner(cfg, agentStore, quotaStore)
opts = append(opts, rest.WithInteractiveSpawner(spawner))
```

### Working reference

**`backend/internal/adapter/cli/dashboard.go:72-73`** — correctly creates spawner and wires it:
```go
spawner := agent.NewSpawner(cfg, store, tracker)
wsSessions := wsAdapter.NewSessionManager()
```

### The fix

Add two lines to `serve.go` after the existing options:
1. Import `agent` package
2. Create `agent.NewSpawner(cfg, agentStore, quotaStore)`
3. Append `rest.WithInteractiveSpawner(spawner)` to opts

`WithInteractiveSpawner` already creates the `wsAdapter.SessionManager` internally (see `server.go:78-81`), so no additional wiring is needed.
