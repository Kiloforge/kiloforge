# Specification: Fix 'crelay add' to Accept Remote URLs

## Summary

Fix `crelay add` to accept a git remote URL (SSH or HTTPS) instead of requiring a local path. The repo name should be derived from the remote URL, not from the local directory name. The command clones the remote into a managed location and registers it.

## Context

Two bugs exist in the current `crelay add` implementation:

1. **Slug derived from directory name** — `filepath.Base(absPath)` uses the local directory name. If the user runs `crelay add .` from a worktree named `main`, the slug becomes "main" and Gitea shows "conductor/main", which is wrong. The slug should come from the repo name in the remote URL.

2. **Only accepts local paths** — The command validates `filepath.Join(absPath, ".git")` exists, so it rejects remote URLs like `git@github.com:user/repo.git` or `https://github.com/user/repo.git`.

## Codebase Analysis

- **`internal/cli/add.go`** — Current implementation at `runAdd()`:
  - Line 46-53: Treats arg as a local path, resolves with `filepath.Abs()`
  - Line 56-59: Validates `.git` directory exists — rejects remote URLs
  - Line 76: Derives slug from `filepath.Base(absPath)` — wrong for worktrees
  - Line 94-97: Detects origin remote from the local repo — not needed when arg IS the remote
  - Line 109-114: Adds `gitea` remote to existing local repo
  - Line 118-124: Pushes `main` branch to Gitea
- **`internal/project/registry.go`** — `Project` struct has `ProjectDir` and `OriginRemote` fields. `ProjectDir` will point to the cloned location.
- **`internal/gitea/client.go`** — `CreateRepo()` and `CreateWebhook()` work with repo names, no changes needed.

### New Add Flow

```
1. Parse <remote-url> arg (required)
2. Derive slug from remote URL (or --name flag)
3. Load global config, verify Gitea is running
4. Check projects.json — error if slug already registered
5. Clone remote into ~/.crelay/repos/<slug>/
6. Create Gitea repo via API
7. Add gitea remote to cloned repo
8. Push main branch to Gitea
9. Register webhook
10. Create project data dir
11. Register in projects.json with OriginRemote = provided URL, ProjectDir = clone path
12. Print success
```

### Managed Clone Directory

Clones go into `~/.crelay/repos/<slug>/`. This keeps all crelay-managed repos in one place and avoids confusion with user worktrees.
