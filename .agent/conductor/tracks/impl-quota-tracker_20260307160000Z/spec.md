## Summary

Parse stream-json output from spawned CC agents to extract token usage, cost metrics, and error events. Aggregate into a centralized in-memory quota tracker with file persistence.

## Context

Crelay spawns multiple CC agents as child processes with `--output-format stream-json`. The stream-json output contains structured events including token usage, cost data, and error information. Currently the spawner goroutine writes this output to log files but does not parse it. We need a tracker that aggregates usage across all active workers to enable quota-aware agent management.

## Codebase Analysis

- **Interception point**: `internal/agent/spawner.go` — goroutines that read stdout line-by-line via `bufio.Scanner`
- **Current behavior**: Lines are written to log file verbatim, no parsing
- **State store**: `internal/state/state.go` — `AgentInfo` struct and `Store` with JSON persistence
- **No existing metrics**: No token/cost tracking anywhere in the codebase

### Stream-JSON format (from CC documentation)

CC stream-json emits newline-delimited JSON objects. Key event types:
- `result` events with `usage` field containing `input_tokens`, `output_tokens`, `cache_read_input_tokens`
- `error` events with status codes (429, 529) and error messages
- Cost can be derived from token counts and model pricing

### OTel alternative

CC supports `CLAUDE_CODE_ENABLE_TELEMETRY=1` with OTLP export, but this requires an external collector. The stream-json parsing approach is simpler and self-contained — no external infrastructure needed.
