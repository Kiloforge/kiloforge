## Summary

Enable automatic Gitea authentication through the orchestrator's reverse proxy. Users accessing Gitea through the orchestrator URL are automatically logged in as the admin user — no password needed. This eliminates credential management for the local Gitea instance.

## Context

Gitea currently requires username/password login, which is unnecessary friction for a local-only development tool. The orchestrator already reverse-proxies all Gitea traffic (`localhost:4001/` → Gitea). Gitea supports reverse proxy authentication where a trusted proxy injects an `X-WEBAUTH-USER` header and Gitea treats the request as authenticated.

This is safe because:
- Gitea is bound to localhost only (not exposed to the network)
- Traffic is HTTP (not HTTPS) — already not secured
- Single-user/single-machine tool — no multi-tenant concerns
- The orchestrator proxy is the intended entry point

Future auth requirements (if ever needed) can be layered on top without removing this.

## Technical Notes

### Gitea Environment Variable

```yaml
- GITEA__service__ENABLE_REVERSE_PROXY_AUTHENTICATION=true
```

This tells Gitea to trust the `X-WEBAUTH-USER` header. The default header name is `X-WEBAUTH-USER` — no need to customize it.

### Proxy Header Injection

```go
func NewGiteaProxy(targetURL, authUser string) http.Handler {
    target, err := url.Parse(targetURL)
    if err != nil {
        panic("invalid gitea proxy target URL: " + err.Error())
    }
    proxy := httputil.NewSingleHostReverseProxy(target)
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        r.Header.Set("X-WEBAUTH-USER", authUser)
        proxy.ServeHTTP(w, r)
    })
}
```

### Security Note

Gitea should only accept the `X-WEBAUTH-USER` header from the trusted proxy, not from external clients. Since Gitea's port (4000) is only used internally by the orchestrator and direct access still requires password auth, this is safe. The `ENABLE_REVERSE_PROXY_AUTHENTICATION` flag doesn't bypass direct login — it only trusts the header when present.
