## Summary

Add a track detail page accessible from both the kanban board and track list. Display the track's specification, implementation plan (rendered as markdown), phase/task progress, metadata, and agent/trace links.

## Context

The dashboard currently shows tracks as cards on a kanban board and as items in a track list, but neither is clickable for viewing details. Rich track data (spec, plan, progress) exists on the backend via the new `GET /api/tracks/{trackId}` endpoint but the frontend has no way to display it. Users need to see what a track entails before approving it or understanding its progress.
