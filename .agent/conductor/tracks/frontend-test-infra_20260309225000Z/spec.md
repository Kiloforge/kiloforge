## Summary

Set up frontend testing infrastructure with Vitest and React Testing Library. Add initial tests for key hooks and components. Wire `npm test` into the Makefile `test` target so `make test` runs both backend Go tests and frontend tests.

## Context

The frontend has zero test infrastructure — no test framework, no test files, no `test` script in `package.json`. The Makefile `test` target only runs `cd backend && go test -race ./...`. Frontend regressions (like the tour API mismatch) go undetected.

## Codebase Analysis

### Current state

- **`frontend/package.json`** — scripts: `dev`, `build` only. No `test` script. No vitest/jest in dependencies.
- **`frontend/src/`** — No `*.test.*` or `*.spec.*` files anywhere.
- **`Makefile`** — `test` target: `cd backend && go test -race ./...`. No frontend test step.
- **Vite config** — `frontend/vite.config.ts` uses React plugin. Vitest integrates natively with Vite.

### Test framework choice

**Vitest** — native Vite integration, same config, fast HMR-aware test runner. Pairs with `@testing-library/react` for component tests and `msw` for API mocking.

### Priority test targets

Based on recent bugs and critical paths:

1. **Hooks** — `useTour`, `useProjects`, `useBoard`, `useConsent` — these bridge FE to BE and are where contract mismatches occur
2. **Components** — `AddProjectForm` (validation), `KanbanBoard` (drag logic), `TourOverlay` (step progression)
3. **API layer** — `queryKeys`, `fetcher` — ensure correct URL construction and error handling

## Technical Notes

### Setup

```bash
cd frontend
npm install -D vitest @testing-library/react @testing-library/jest-dom @testing-library/user-event jsdom msw
```

### Vitest config

```ts
// vite.config.ts — add test section
export default defineConfig({
  plugins: [react()],
  base: '/',
  test: {
    environment: 'jsdom',
    globals: true,
    setupFiles: './src/test/setup.ts',
  },
});
```

### Makefile update

```makefile
test: ensure-dist
	cd backend && go test -race ./...
	cd frontend && npm test -- --run

test-all: ensure-dist
	cd backend && go test -race -tags=integration ./...
	cd frontend && npm test -- --run
```
