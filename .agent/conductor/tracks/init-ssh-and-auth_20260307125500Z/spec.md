# Specification: SSH Key Auto-Registration and Randomized Admin Password

**Track ID:** init-ssh-and-auth_20260307125500Z
**Type:** Feature
**Created:** 2026-03-07T12:55:00Z
**Status:** Draft

## Summary

During `crelay init`, auto-register the user's SSH public key with the Gitea conductor user for git-over-SSH support. Generate a random admin password unless explicitly provided via `--admin-pass` flag or already saved in config from a previous init. Store the password in global config for reuse.

## Context

Currently the Gitea admin password is hardcoded as `conductor123` in `internal/config/config.go`. This is insecure even for local use. SSH key registration enables git-over-SSH which is more natural for developers than HTTP basic auth URLs. Both improvements make the init flow more production-ready.

## Codebase Analysis

- **`internal/config/config.go`** — `GiteaAdminPass = "conductor123"` is a package-level constant. Needs to become a config field resolved via the adapter chain.
- **`internal/gitea/manager.go:Configure()`** — Creates admin user with `config.GiteaAdminPass`. Uses `config.GiteaAdminUser` and `config.GiteaAdminEmail`.
- **`internal/gitea/client.go`** — `NewClient()` takes username/password. Multiple call sites use `config.GiteaAdminPass`.
- **`internal/cli/init.go`** — Prints admin credentials on success. Would need to show generated password.
- **SSH key location** — Typically `~/.ssh/id_rsa.pub`, `~/.ssh/id_ed25519.pub`, or similar.
- **Gitea SSH key API** — `POST /api/v1/user/keys` with `{"title": "...", "key": "ssh-ed25519 ..."}`.

## Acceptance Criteria

- [ ] Admin password resolution: `--admin-pass` flag > saved config (`GiteaAdminPass` field) > generate random
- [ ] Generated password is cryptographically random (16+ chars, alphanumeric)
- [ ] Password stored in global config after generation for reuse on subsequent `init` runs
- [ ] Re-running `init` with existing config reuses the saved password (does not generate new one)
- [ ] `--admin-pass` flag overrides saved config and updates it
- [ ] User's SSH public key auto-detected from `~/.ssh/` (try `id_ed25519.pub`, `id_rsa.pub`, `id_ecdsa.pub`)
- [ ] SSH key registered with Gitea conductor user via API (`POST /api/v1/user/keys`)
- [ ] If no SSH key found, warn but continue (not a fatal error)
- [ ] `--ssh-key` flag allows specifying a custom SSH public key path
- [ ] Init success output shows the admin password (so user can access Gitea web UI)
- [ ] Remove hardcoded `GiteaAdminPass` constant

## Dependencies

None (can be implemented independently, modifies init flow)

## Out of Scope

- SSH-based git remote URLs (currently HTTP, SSH remotes are a future enhancement)
- Multi-user SSH key management
- SSH key rotation

## Technical Notes

### Password Resolution

```go
func resolveAdminPass(flagValue string, cfg *config.Config) string {
    if flagValue != "" {
        return flagValue  // explicit flag wins
    }
    if cfg.GiteaAdminPass != "" {
        return cfg.GiteaAdminPass  // reuse saved
    }
    return generateRandomPassword(20)  // generate new
}
```

### SSH Key Detection Order

1. `~/.ssh/id_ed25519.pub` (preferred, modern)
2. `~/.ssh/id_rsa.pub` (common, legacy)
3. `~/.ssh/id_ecdsa.pub` (less common)
4. Custom path via `--ssh-key` flag

### Gitea SSH Key API

```go
func (c *Client) AddSSHKey(ctx context.Context, title, pubKey string) error {
    payload := map[string]any{
        "title": title,
        "key":   pubKey,
    }
    _, err := c.do(ctx, "POST", "/api/v1/user/keys", payload)
    return err
}
```
