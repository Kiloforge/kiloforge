## Summary

Implement `crelay implement <track-id>` to approve a track and spawn a Claude Code developer agent in a pooled worktree. The agent implements the track, creates a PR on Gitea, tags it with session/track metadata, and pauses. This is the entry point for the agent orchestration lifecycle.

## Context

See `docs/design-agent-orchestration.md` Sections 3-4. The implement command bridges the conductor track system with crelay's agent spawning. It acquires a worktree from the pool, spawns a developer agent, and records state for the relay to manage the review cycle.

- [ ] Agent spawned with a generated session ID, output streamed to log file

- [ ] Error if worktree pool is exhausted
- [ ] `--project` flag to specify which registered project to implement in

## Technical Notes

### Flow
```
1. Validate track exists and is pending/approved
2. Resolve project (--project flag or auto-detect from cwd)
3. Acquire worktree from pool
4. Prepare worktree: reset to main, create branch <track-id>
5. Spawn claude -p "/conductor-developer <track-id>" --session-id <uuid>
6. Record agent state
7. Print agent info and log file path
8. Command returns (agent runs in background)
```

### Agent Spawning
Extends `internal/agent/spawner.go` to accept worktree path and track context. The spawner sets working directory to the acquired worktree.
