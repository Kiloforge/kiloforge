## Summary

Remove the `-buildvcs=false` workaround and enable proper VCS stamping in git worktrees by setting `GIT_DIR` and `GIT_WORK_TREE` environment variables during build. This embeds commit hash, timestamp, and dirty flag into the binary.

## Context

The build currently uses `-buildvcs=false` because Go's VCS detection fails in git worktrees — the `.git` entry in a worktree is a file (pointer), not a directory, which confuses the Go toolchain. The fix is to set `GIT_DIR=$(git rev-parse --git-common-dir)` and `GIT_WORK_TREE` to the worktree root before building. This has been verified to work — the binary gets `vcs.revision`, `vcs.time`, and `vcs.modified` stamps.

## Technical Notes

### Why GIT_DIR + GIT_WORK_TREE works

In a worktree, `.git` is a file containing `gitdir: /path/to/.git/worktrees/<name>`. Go's VCS detection calls `git status` which needs the actual `.git` directory. Setting `GIT_DIR` to `$(git rev-parse --git-common-dir)` (the shared `.git` dir) and `GIT_WORK_TREE` to the worktree root gives git the correct context.

This works identically in non-worktree checkouts: `git rev-parse --git-common-dir` returns `.git` and `GIT_WORK_TREE` is the repo root — same as the default.
