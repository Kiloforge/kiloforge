# Specification: Interactive SSH Key Selection for Add Command

## Summary

When `kf add` receives an SSH-based remote URL and no `--ssh-key` flag is provided, interactively discover available SSH keys in `~/.ssh/` and prompt the user to select one.

## Context

Currently, `kf add` supports `--ssh-key <path>` to specify an SSH private key for git operations. If the flag is omitted and the remote is SSH-based (`git@...`), the clone relies on the user's default SSH agent configuration. This is fragile — users with multiple keys (personal GitHub, work GitLab, deploy keys, etc.) may hit authentication failures or use the wrong key silently.

## Codebase Analysis

- **`backend/internal/adapter/cli/add.go`**: The `--ssh-key` flag is defined at line 46. Lines 72-85 resolve the path and build `GIT_SSH_COMMAND` env. If the flag is empty, no SSH key handling occurs.
- **`backend/internal/adapter/auth/sshkey.go`**: `DetectSSHKey()` already searches `~/.ssh/` for keys in preference order (`id_ed25519`, `id_rsa`, `id_ecdsa`) but returns only the first match. No multi-key discovery or selection.
- **`domain.Project.SSHKeyPath`**: Already stored per-project and used by `GitSSHEnv()` for subsequent git operations.
- **`isRemoteURL()` in add.go**: Already detects SSH URLs (`git@host:path` format) — can be used to trigger the selection prompt.

## Acceptance Criteria

- [ ] When `kf add git@...` is called without `--ssh-key`, discover all SSH keys in `~/.ssh/`
- [ ] Present a numbered list of discovered keys and prompt user to select one (or skip)
- [ ] Selected key is used for the clone and stored in the project's `SSHKeyPath`
- [ ] If only one key exists, auto-select it with a confirmation message (no prompt)
- [ ] If no keys are found, warn and proceed without SSH key configuration
- [ ] `--ssh-key` flag still works as before (bypasses interactive selection)
- [ ] HTTPS remotes skip SSH key selection entirely
- [ ] Non-interactive mode (e.g., piped stdin) falls back to auto-detect or skips gracefully

## Technical Notes

### Implementation Approach

1. **Extend `auth.sshkey.go`** — add `DiscoverSSHKeys(sshDir string) []SSHKeyInfo` that returns all found keys with metadata (name, type, fingerprint, path).

2. **Add selection prompt in `add.go`** — after detecting an SSH remote URL and no `--ssh-key` flag:
   ```
   SSH remote detected. Select an SSH key for git operations:
     1) ~/.ssh/id_ed25519 (ED25519 SHA256:abc...)
     2) ~/.ssh/id_rsa (RSA SHA256:def...)
     3) Skip — use default SSH agent
   Select [1]:
   ```

3. **Auto-select single key** — if only one key exists, print "Using SSH key: ~/.ssh/id_ed25519" and proceed.

4. **Read from stdin** — use `bufio.Scanner` on `os.Stdin` for the prompt. Detect non-interactive with `term.IsTerminal()` or check if stdin is a pipe.
