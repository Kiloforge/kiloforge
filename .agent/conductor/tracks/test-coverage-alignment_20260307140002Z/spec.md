## Summary

Fill test coverage gaps to align with the Go style guide testing requirements. Create shared testutil package, add tests for untested packages, improve error path coverage, and adopt table-driven test patterns consistently.

## Context

The style guide requires:
- Every non-trivial package has tests
- Error paths are tested for every function that returns an error
- Table-driven tests for parametric scenarios
- Shared testutil package with mock implementations
- `t.Parallel()` where safe
- HTTP handler tests verify no internal error leakage

Current gaps:
- `agent/` and `state/` have zero tests
- No shared testutil package — mocks duplicated across test files
- Many packages lack error path tests
- Inconsistent use of table-driven tests and `t.Parallel()`

### Error path test requirements per layer

**Domain:** Validation failures return specific errors
**Service:** Propagates port errors correctly, handles missing entities
**Adapter/persistence:** Corrupt JSON, missing files, permission errors
**Adapter/rest:** 400 for bad input, 404 for not-found, 500 for internal errors, no detail leakage
