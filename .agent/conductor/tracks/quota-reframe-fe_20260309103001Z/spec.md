## Summary

Update the dashboard UI to display token usage as the primary metric and reframe USD cost as secondary informational data ("estimated API-equivalent cost"). Aligns the frontend with the backend API schema changes from `quota-reframe-be_20260309103000Z`.

## Context

The current dashboard prominently displays "Total Cost: $X.XX" as a primary stat card, implying a billing event. Users are on Claude Code subscriptions — they don't pay per-token. The real value proposition is showing token consumption patterns and rate limit status. The cost should be reframed as "Estimated API Cost" — informational data showing what the work would cost on the API (nice marketing for subscription value).

## Technical Notes

### StatCards Layout Change

**Before (cost-primary):**
```
[Agents: 3] [Total Cost: $4.23] [Rate Limit: OK] [Tokens: 125k / 12k]
```

**After (tokens-primary):**
```
[Agents: 3] [Tokens: 125k in / 12k out] [Rate Limit: OK] [Est. API Cost: $4.23]
```

The token card should show all four token types when cache tokens are non-zero:
- Input tokens
- Output tokens
- Cache read tokens (if > 0)
- Cache creation tokens (if > 0)

### AgentCard Enhancement

Per-agent display should shift from cost-first to tokens-first:
- Primary: token counts (input/output)
- Secondary: cache tokens (if present)
- Tertiary: estimated API cost (dimmed/smaller)
