# Specification: Auto-Initialize Conductor Artifacts Before Track Generation

**Track ID:** fix-track-gen-project-init_20260309235500Z
**Type:** Bug
**Created:** 2026-03-09T23:55:00Z
**Status:** Draft

## Summary

When generating tracks from the dashboard for a project that hasn't been initialized with conductor/kiloforge artifacts, Claude exits immediately with exit 0 because the `/kf-track-generator` skill halts at pre-flight ("not initialized — run `/kf-setup` first"). The dashboard should either auto-initialize the project or chain setup before track generation.

## Context

A freshly added project (e.g., `example-project` cloned from GitHub) has no `.agent/conductor/` directory. The track generator skill's Step 1 checks for `product.md`, `tech-stack.md`, and `index.md` — if missing, it halts. This means:

1. User adds project via dashboard
2. User clicks "Generate Tracks" with a prompt
3. Claude spawns, runs `/kf-track-generator`, immediately detects no conductor artifacts
4. Claude outputs "not initialized" message and exits
5. Dashboard shows "exit 0" with no useful feedback

The user sees the agent terminal "trying to connect" and then "exit 0" with no explanation.

## Codebase Analysis

### Track generator pre-flight

- **`backend/internal/adapter/skills/embedded/kf-track-generator/SKILL.md`** — Step 1 checks:
  ```bash
  git show main:.agent/conductor/product.md > /dev/null 2>&1
  git show main:.agent/conductor/tech-stack.md > /dev/null 2>&1
  git show main:.agent/conductor/index.md > /dev/null 2>&1
  ```
  If missing: "Display error and suggest running `/conductor-setup` first. **HALT.**"

### GenerateTracks API handler

- **`backend/internal/adapter/rest/api_handler.go:1237-1294`** — spawns interactive agent with prompt `/kf-track-generator ...`
- No pre-check for conductor artifacts before spawning
- `workDir` is set to `ProjectDir` (the cloned repo at `~/.kiloforge/repos/{slug}`)

### Setup skill

- **`backend/internal/adapter/skills/embedded/kf-setup/SKILL.md`** — initializes conductor artifacts (product.md, tech-stack.md, index.md, workflow.md, tracks.md)

### The fix options

**Option A (preferred): Chain setup into the prompt**
- If conductor artifacts don't exist in the project dir, prefix the prompt with `/kf-setup` followed by `/kf-track-generator`
- Or: modify the track generator skill to auto-run setup if artifacts are missing (instead of halting)

**Option B: Pre-check in API handler**
- Before spawning, check if `.agent/conductor/product.md` exists in `workDir`
- If missing, return a specific error telling the user to initialize first
- Less ideal because it adds another error state the user must manually resolve

**Option C: Auto-initialize in API handler**
- Before spawning the track gen agent, run a quick setup agent first
- More complex, requires sequential agent spawning

## Technical Notes

### Recommended approach: Modify the track generator skill

Change Step 1 in `kf-track-generator/SKILL.md` from "HALT if not initialized" to "auto-run setup if not initialized":

```
### Step 1 — Verify Conductor is initialized

If missing, automatically run the setup workflow:
1. Create `.agent/conductor/` directory
2. Generate minimal `product.md`, `tech-stack.md`, `index.md`, `tracks.md` by analyzing the codebase
3. Continue with track generation
```

This is the cleanest fix because:
- No API changes needed
- Works for both dashboard and CLI track generation
- The skill already has access to the codebase context to generate meaningful setup artifacts

### Alternative: Modify the prompt in GenerateTracks

```go
// Check if conductor artifacts exist
conductorDir := filepath.Join(workDir, ".agent", "conductor")
if _, err := os.Stat(filepath.Join(conductorDir, "product.md")); os.IsNotExist(err) {
    fullPrompt = fmt.Sprintf("/kf-setup && /kf-track-generator %s", req.Body.Prompt)
}
```

This chains the setup skill before track generation in the prompt.
