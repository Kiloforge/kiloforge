## Summary

Fix the dashboard showing 0 tokens and $0.00 estimated cost for agents by bridging the in-memory `QuotaTracker` (which receives SDK cost/token events) to the REST API and SSE event system. Currently two separate quota systems exist that never communicate.

## Context

After talking to an agent, the dashboard still shows 0 tokens and $0.00 cost. The root cause is a disconnect between two quota systems:

1. **QuotaTracker** (in-memory + `quota-usage.json` on disk) — receives `RecordEvent()` calls from `sdk_client.relayResponse()` with real cost and token data
2. **QuotaStore** (SQLite `quota_usage` table) — queried by the REST API and dashboard watcher, but `RecordUsage()` is never called by anything

The spawner gets `quotaTracker`, the dashboard/API gets `quotaStore`. Data flows into the tracker but the API reads from the store — they never meet.

## Codebase Analysis

- **`serve.go` (lines 84, 103, 119-122)**: Creates `quotaStore` (SQLite) and `quotaTracker` (JSON) separately. Dashboard gets `quotaStore`, spawner gets `quotaTracker`
- **`sdk_client.go` (lines 140-143)**: `tracker.RecordEvent(agentID, ev)` correctly records to `QuotaTracker`
- **`api_handler.go` (lines 815-822)**: `domainAgentToGen()` calls `quota.GetAgentUsage(a.ID)` where `quota` is the SQLite `QuotaStore` — returns nil/0 because nothing writes to it
- **`QuotaStore.RecordUsage()`** (`sqlite/quota_store.go`): Defined but never called anywhere in the codebase
- **`watcher.go` (lines 69-81)**: Dashboard watcher reads from `quotaStore` for SSE quota_update events — always 0
- **`StatCards.tsx`**: Frontend correctly displays `quota?.estimated_cost_usd` — but receives 0 from API

## Technical Notes

### Recommended Fix: Option A — Pass QuotaTracker to API

The simplest fix: make `QuotaTracker` implement `QuotaReader` interface (or whatever the API handler expects) and pass it directly instead of `QuotaStore`. This eliminates the disconnect entirely.

1. Check if `QuotaTracker` already satisfies the `QuotaReader` interface used by `domainAgentToGen`
2. If yes: change `serve.go` to pass `quotaTracker` to the dashboard/API instead of `quotaStore`
3. If no: add the missing method(s) to `QuotaTracker`
4. Remove the unused `QuotaStore` SQLite implementation (or keep it for future migration)
5. Update the watcher to read from `quotaTracker`

### Alternative: Option B — Bridge via RecordUsage

Call `quotaStore.RecordUsage()` from the tracker's `RecordEvent()` method. This requires passing the store to the tracker or using a callback. More complex and still has two sources of truth.
