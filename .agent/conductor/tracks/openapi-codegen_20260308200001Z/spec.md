## Summary

Adopt schema-first API design using OpenAPI 3.0 specs and `oapi-codegen` to replace hand-written `HandleFunc` routes with generated server stubs, models, and client code.

## Context

The relay server currently uses hand-written `http.HandleFunc` registrations with manual request parsing and response writing. This is error-prone, lacks type safety, and makes it hard to keep API documentation in sync with implementation. The lmf-wt project demonstrates a mature pattern using `oapi-codegen` with `StrictServerInterface` that we should adopt.

## Codebase Analysis

- **Current routes** (from `rest/server.go`): `/webhook`, `/health`, plus dashboard routes at `/-/api/agents`, `/-/api/quota`, `/-/api/tracks`, `/-/api/status`, `/-/events` (SSE), badge routes at `/-/api/badges/*`, lock routes at `/-/api/locks*`.
- **Handler pattern**: Raw `http.HandleFunc` with manual JSON marshal/unmarshal, manual path parameter extraction via `r.PathValue()`.
- **No existing OpenAPI spec** — must be authored from scratch based on current API surface.
- **Reference implementation** (lmf-wt): Uses `oapi-codegen` v2.5.1, `cfg.yaml` config files, `StrictServerInterface` pattern with Fiber adapter, `StrictMiddlewareFunc` for auth.
- **Badge/SSE routes** are special — badges return SVG, SSE returns event stream. These may need to stay as manual handlers alongside generated code.
- **Lock routes** use REST-style paths (`/-/api/locks/{id}/acquire`).

## Technical Notes

### oapi-codegen setup
```yaml
# api/cfg.yaml
package: gen
output: internal/adapter/rest/gen/server.gen.go
generate:
  strict-server: true
  models: true
  client: true
```

### Migration strategy
1. Author OpenAPI spec from existing API surface
2. Generate strict server interface + models
3. Implement the interface in a new handler struct
4. Wire generated router into the existing mux alongside manual routes (webhook, SSE, badges)
5. Gradually move manual handlers to generated ones
6. Remove old hand-written handler code

### Special endpoints
- **Webhook**: Stays manual — payloads are Gitea-defined, not our API contract
- **SSE (`/-/events`)**: Stays manual — streaming responses aren't well-modeled in OpenAPI
- **Badges (`/-/api/badges/*`)**: Stays manual — returns SVG, not JSON

### Makefile targets
```makefile
gen-api:
	oapi-codegen -config api/cfg.yaml api/openapi.yaml

verify-codegen:
	oapi-codegen -config api/cfg.yaml api/openapi.yaml > /tmp/gen.go
	diff internal/adapter/rest/gen/server.gen.go /tmp/gen.go
```
