# Specification: Fix Nested Claude Session Detection — Unset CLAUDECODE Env Var

**Track ID:** fix-nested-claude-env_20260310001000Z
**Type:** Bug
**Created:** 2026-03-10T00:10:00Z
**Status:** Draft

## Summary

All agent spawning fails silently when Kiloforge runs inside a Claude Code session. Claude detects the `CLAUDECODE` environment variable from the parent process and refuses to start: "Claude Code cannot be launched inside another Claude Code session." The spawner inherits the parent env without cleaning it, so the child `claude` process exits immediately with no stdout output.

## Context

This is the **root cause** of the "exit 0" issue when generating tracks from the dashboard. The agent process inherits `CLAUDECODE` from the parent environment. Claude CLI checks this env var and aborts to prevent nested sessions. Since `cmd.Stderr` goes to a log file and `cmd.Stdout` is piped for stream-json parsing, the error message is only visible in stderr (log file) — but it's actually printed to the shell's stderr before the pipe is set up, so even the log file is 0 bytes.

This affects ALL agent spawning methods:
- `SpawnInteractive` (interactive agents, track generation, admin operations)
- `SpawnDeveloper` (via `server.go` defaultSpawner)
- `SpawnReviewer` (via `server.go` defaultSpawner)

## Codebase Analysis

### Root cause

- **`backend/internal/adapter/agent/spawner.go:377`** — `cmd := exec.CommandContext(ctx, "claude", args...)` inherits parent env
- No `cmd.Env` is set, so `CLAUDECODE` (and any other Claude-internal env vars) propagate to child
- Claude CLI checks `CLAUDECODE` and exits with error before producing any stream-json output

## Technical Notes

### The fix

Create a helper in `spawner.go`:

```go
// cleanClaudeEnv returns os.Environ() with Claude-internal env vars removed
// to prevent "nested session" detection in child claude processes.
func cleanClaudeEnv() []string {
    var env []string
    for _, e := range os.Environ() {
        if strings.HasPrefix(e, "CLAUDECODE=") {
            continue
        }
        env = append(env, e)
    }
    return env
}
```

Then in each spawner, before `cmd.Start()`:
```go
cmd.Env = cleanClaudeEnv()
```
