# Specification: Fix Agent Spawner — Add --verbose Flag for stream-json

## Summary

All agent spawning exits silently with code 0 because Claude CLI now requires `--verbose` when combining `-p` (print mode) with `--output-format stream-json`. Without it, Claude prints an error to stderr and exits immediately — producing 0 bytes of stdout, which the spawner interprets as a clean exit.

## Context

This is the **actual root cause** of the persistent "Agent exited (code 0)" issue. The previous fix (track `fix-nested-claude-env`) correctly addressed the `CLAUDECODE` env var, but the problem persisted because of a separate Claude CLI requirement change: `--output-format=stream-json` now requires `--verbose` when used with `--print` (`-p`).

Verified by running:
```
$ claude -p "say hello" --output-format stream-json 2>&1
Error: When using --print, --output-format=stream-json requires --verbose
```

Adding `--verbose` resolves it:
```
$ claude -p "say hello" --output-format stream-json --verbose 2>&1 | head -1
{"type":"system","subtype":"hook_started",...}
```
