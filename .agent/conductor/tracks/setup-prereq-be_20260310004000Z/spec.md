# Specification: Setup Prerequisite Check Before Agent Spawning (Backend)

**Track ID:** setup-prereq-be_20260310004000Z
**Type:** Feature
**Created:** 2026-03-10T00:40:00Z
**Status:** Draft

## Summary

Make conductor setup (`kf-setup`) a blocking prerequisite before any agent work can happen on a project. Currently, `GenerateTracks` silently chains `/kf-setup` into the prompt, but setup asks 17-20 interactive questions that cannot work in `-p` (print) mode. Instead, the backend should detect missing setup and return a specific error code so the frontend can guide the user through an interactive setup session first.

## Context

The `kf-setup` skill is an interactive Q&A process that generates conductor artifacts (`product.md`, `tech-stack.md`, `workflow.md`, etc.). Without these artifacts, track generation and developer agents cannot function — they rely on product context, tech stack info, and the tracks registry.

The current auto-chain approach (`/kf-setup\n\n...` prepended to the track gen prompt) fails because:
1. `-p` mode doesn't support interactive Q&A — the agent can't ask the user questions
2. The agent either makes up answers or halts at the first question
3. The user never sees the setup questions, resulting in garbage or missing artifacts

### New endpoint: `GET /api/projects/{slug}/setup-status`

```go
func (h *APIHandler) GetProjectSetupStatus(ctx context.Context, req gen.GetProjectSetupStatusRequestObject) (gen.GetProjectSetupStatusResponseObject, error) {
    project, ok := h.findProject(req.Slug)
    if !ok {
        return gen.GetProjectSetupStatus404JSONResponse{Error: "project not found"}, nil
    }
    workDir := project.WorktreeDir
    productPath := filepath.Join(workDir, ".agent", "conductor", "product.md")
    _, err := os.Stat(productPath)
    return gen.GetProjectSetupStatus200JSONResponse{
        SetupComplete: err == nil,
        ProjectSlug:   req.Slug,
    }, nil
}
```

### New endpoint: `POST /api/projects/{slug}/setup`

Spawns an interactive agent with `/kf-setup` as the prompt. Uses the same `SpawnInteractive` + WebSocket bridge pattern as `SpawnInteractiveAgent`.

### Setup check helper

```go
func (h *APIHandler) checkSetup(projectSlug string) *string {
    project, ok := h.findProject(projectSlug)
    if !ok {
        return nil // no project context = no setup needed
    }
    productPath := filepath.Join(project.WorktreeDir, ".agent", "conductor", "product.md")
    if _, err := os.Stat(productPath); os.IsNotExist(err) {
        msg := "kiloforge setup required"
        return &msg
    }
    return nil
}
```

### HTTP 428 Precondition Required

Using 428 (RFC 6585) — "the server requires the request to be conditional." This fits semantically: the operation requires setup to be completed first. The response body includes the project slug so the frontend knows which project needs setup.
