## Summary

Change the default for `TracingEnabled` from `false` to `true`, add environment variable support (`KF_TRACING_ENABLED`), and expose a REST endpoint to toggle tracing at runtime.

## Context

Tracing is currently disabled by default and can only be enabled by editing `~/.kiloforge/config.json`. This makes it invisible to most users. Since tracing is lightweight (in-memory store + optional OTLP export) and provides valuable debugging insight, it should be on by default. A runtime toggle via the API allows the dashboard to control it without restarting.

### Config API endpoints
```yaml
GET /api/config
  → Returns: { tracing_enabled: bool, dashboard_enabled: bool }

PUT /api/config
  → Body: { tracing_enabled?: bool, dashboard_enabled?: bool }
  → Persists to config.json
```
