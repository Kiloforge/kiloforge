## Summary

Fix the recurring bug where `kf init` displays a blank password after first-time setup. The root cause is in `manager.go` line 85 which clears `cfg.GiteaAdminPass` on the shared config pointer before `init.go` gets to display it. Previous fixes addressed the wrong layers.

## Context

This bug has been "fixed" twice:
1. **remove-password-persistence_20260308210000Z** — Fixed `json_adapter.Save()` to strip password using a copy (correct, still in place)
2. **fix-init-password-display_20260308235500Z** — Fixed the early-exit path to conditionally display password (correct, still in place)

Neither fix addressed the **actual root cause**: `manager.Configure()` mutates the shared config pointer by clearing `GiteaAdminPass` at line 85 of `manager.go`, which runs BEFORE `init.go` line 146 tries to display it.

The comment on line 83 says "Clear password so it is not persisted to config.json" — but `json_adapter.Save()` already handles this by making a copy and clearing the copy. The mutation in manager.go is both **redundant** and **destructive**.

## The Fix

**Remove lines 83-85 from `manager.go`:**
```go
// DELETE THESE LINES:
// Clear password so it is not persisted to config.json (omitempty removes it).
// The password was only needed for admin user creation and token generation above.
m.cfg.GiteaAdminPass = ""
```

**Why this is safe:**
- `json_adapter.Save()` (lines 40-48) already creates a copy of the config, clears the password on the copy, and marshals the copy. The original config is never mutated during save.
- The `omitempty` tag on `GiteaAdminPass` was the original safety mechanism, but `json_adapter.Save()` now explicitly strips it regardless.
- After init.go displays the password at line 146, the config object goes out of scope — the password is never stored.

**Alternative fix (defensive):** Keep manager.go's clear but save the password to a local variable in init.go before calling Configure:
```go
generatedPassword := cfg.GiteaAdminPass
client, err = manager.Configure(ctx)
// ... later ...
fmt.Printf("  Admin: %s / %s\n", cfg.GiteaAdminUser, generatedPassword)
```

**Recommendation:** Remove lines 83-85 from manager.go. It's the cleaner fix — the manager shouldn't mutate caller state for a concern (persistence) that's already handled by the config layer.
