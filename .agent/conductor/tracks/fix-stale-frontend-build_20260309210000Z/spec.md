# Specification: Fix Stale Frontend Build — Rebuild dist with Correct Base Path

**Type:** Bug
**Created:** 2026-03-09T21:00:00Z

## Summary

The dashboard is broken because the embedded `dist/` assets were never rebuilt after `vite.config.ts` was fixed from `base: '/-/'` to `base: '/'`. The stale `index.html` references `/-/assets/...` paths that don't exist, causing the SPA catch-all to serve `index.html` (text/html) instead of JS/CSS files.

## Context

The `fix-spa-and-init-output` track (completed) updated `vite.config.ts` to use `base: '/'` but never ran `npm run build` to regenerate the `dist/` folder. The embedded `dist/index.html` still contains:

```html
<script type="module" crossorigin src="/-/assets/index-RjAkMtuu.js"></script>
<link rel="stylesheet" crossorigin href="/-/assets/index-Dt3PJskE.css">
```

When the browser requests `GET /-/assets/index-RjAkMtuu.js`, the `spaFileServer` in `embed.go` tries to open `-/assets/index-RjAkMtuu.js` which doesn't exist in the embedded FS. It falls back to serving `index.html` with `Content-Type: text/html`. The browser rejects this as invalid JavaScript and the entire SPA fails to load.

## Codebase Analysis

### Root cause chain

1. `frontend/vite.config.ts` — `base: '/'` (correct, fixed in commit aad0eb3)
2. `backend/internal/adapter/dashboard/dist/index.html` — still references `/-/assets/...` (stale build from when `base` was `'/-/'`)
3. `backend/internal/adapter/dashboard/embed.go` — `spaFileServer` falls back to index.html when file not found (correct behavior for SPA routing, but serves wrong content for missing assets)

### Fix

Run `cd frontend && npm run build` to regenerate `dist/` with correct `/assets/...` paths. The Vite build outputs to `../backend/internal/adapter/dashboard/dist/` (configured in `vite.config.ts` `build.outDir`).

### Prevention

Add a build verification step: after building, check that `dist/index.html` does not contain `/-/` references. This could be a simple grep in the build script or a pre-commit check.
