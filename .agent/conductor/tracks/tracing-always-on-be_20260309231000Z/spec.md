## Summary

Remove the `TracingEnabled` config field and always initialize OpenTelemetry tracing. Tracing is already default-on but currently toggleable via config API and env var — this removes the toggle entirely.

## Context

Tracing was added as opt-in with a toggle to let users disable it if it caused issues. In practice it's always on (defaults to `true` when nil) and there's no reason to disable it. The toggle adds unnecessary config complexity and conditional initialization logic.
