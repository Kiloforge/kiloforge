# Specification: PR Merge, Worktree Cleanup, and Agent Teardown

## Summary

After a PR is approved, the developer agent merges it via the Gitea API, resets the worktree to main, deletes the implementation branch (local and remote), posts a final comment, and returns the worktree to the pool. Both developer and reviewer agents are terminated and their state cleaned up.

## Context

See `docs/design-agent-orchestration.md` Section 6. This is the final stage of the orchestration lifecycle. Clean teardown is critical — worktrees must be returned to the pool, branches deleted, and agent processes stopped so resources are freed for the next track.

## Acceptance Criteria

- [ ] Developer merges PR via Gitea API (`POST /repos/{owner}/{repo}/pulls/{num}/merge`)
- [ ] Merge method configurable (merge commit, rebase, squash — default: merge commit)
- [ ] After merge: worktree reset to main (`git checkout <pool-branch> && git reset --hard main`)
- [ ] Implementation branch deleted locally (`git branch -D <track-id>`)
- [ ] Implementation branch deleted on Gitea remote (`git push gitea --delete <track-id>`)
- [ ] Developer posts final comment on merged PR: "Merge complete. Cleanup done."
- [ ] Worktree returned to pool (status=idle)

## Technical Notes

### Merge API

```go
func (c *Client) MergePR(ctx context.Context, repo string, prNum int, method string) error {
    payload := map[string]any{
        "Do": method,  // "merge", "rebase", "squash"
    }
    _, err := c.do(ctx, "POST",
        fmt.Sprintf("/api/v1/repos/%s/%s/pulls/%d/merge", c.username, repo, prNum),
        payload)
    return err
}
```

### Cleanup Sequence

```
1. Merge PR via API
2. Wait for merge confirmation (poll PR status)
3. Post final comment
4. Worktree cleanup:
   a. git checkout <pool-branch>
   b. git reset --hard main
   c. git branch -D <track-id>
   d. git push gitea --delete <track-id>
5. Return worktree to pool
6. Update track status: complete
7. Update PR tracking: merged
8. Terminate developer agent process
```
