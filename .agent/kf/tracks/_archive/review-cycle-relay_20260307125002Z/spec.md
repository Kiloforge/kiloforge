# Specification: Developer-Reviewer Relay Cycle

**Track ID:** review-cycle-relay_20260307125002Z
**Type:** Feature
**Created:** 2026-03-07T12:50:02Z
**Status:** Draft

## Summary

Implement the webhook-driven orchestration cycle between developer and reviewer agents. When a PR is opened, spawn a reviewer. When the reviewer approves, resume the developer for merge. When changes are requested, resume the developer for revisions. Track review cycle count with a configurable max (default 3) and escalate to human intervention when exceeded.

## Context

See `docs/design-agent-orchestration.md` Sections 4-5. This is the core relay logic that makes kiloforge an orchestrator rather than just a spawner. The relay mediates the conversation between developer and reviewer through Gitea PRs and webhooks.

## Acceptance Criteria

- [ ] PR opened → reviewer agent spawned automatically
- [ ] Reviewer approves → developer agent resumed with "merge" instruction
- [ ] Reviewer requests changes → developer agent resumed with "revise" instruction and review comments
- [ ] Developer pushes revisions → PR updated → reviewer spawned again
- [ ] Review cycle counter tracked per PR (incremented on each revision round)
- [ ] When cycle count >= max (default 3): PR labeled `needs-human-review`, agents stopped, comment posted
- [ ] Reviewer session ID recorded in PR tracking
- [ ] Developer session ID used for `claude --resume` on developer resume
- [ ] `kf escalated` command shows PRs that hit the cycle limit
- [ ] Configurable max review cycles via config or flag

## Technical Notes

### Webhook Event → Action Mapping

| Event | Action | kiloforge Response |
|-------|--------|----------------|
| `pull_request.opened` | PR created by developer | Spawn reviewer, mark dev as waiting-review |
| `pull_request_review` (approved) | Reviewer approved | Resume developer for merge |
| `pull_request_review` (changes_requested) | Reviewer wants changes | Check cycle count, resume dev or escalate |
| `pull_request.synchronize` | Developer pushed revisions | Mark dev as waiting-review, spawn reviewer |

### Escalation

```go
func (s *Server) escalatePR(ctx context.Context, pr *PRTracking, project *project.Project) {
    // Label PR
    s.client.AddLabel(ctx, project.RepoName, pr.PRNumber, "needs-human-review")
    // Comment
    s.client.CommentOnPR(ctx, project.RepoName, pr.PRNumber,
        "Review cycle limit reached (%d). Human review required.", pr.MaxReviewCycles)
    // Stop all agents
    s.stopAgentsForPR(pr)
    // Update tracking
    pr.Status = "escalated"
    pr.Save()
}
```
