# Specification: Branch Diff View with Agent Discuss Button (Frontend)

## Summary

Add a diff viewer to the dashboard that shows file-by-file unified diffs for any agent's working branch vs main, with a "Discuss" button that opens the agent terminal for direct feedback. Also add the missing role badge to the AgentTerminal modal header.

## Context

The diff viewer replaces Gitea's PR review UI, providing a lightweight in-dashboard way to review agent work. Users can see what an agent changed, then click "Discuss" to talk to the agent about those changes via the existing WebSocket terminal.

## Codebase Analysis

- **AgentCard** (`components/AgentCard.tsx`) already shows role badges — the modal needs the same treatment
- **AgentTerminal** (`components/AgentTerminal.tsx`) shows only agent ID in header, no role badge
- **No diff viewer** exists — need a new component
- **CSS Modules** with custom properties — no UI framework, all custom styled
- **TanStack Query v5** for data fetching — new endpoints will use `useQuery`
- **react-diff-viewer-continued** is a good fit — React component for unified/split diffs with syntax highlighting
- **AgentDetailPage** exists at `/agents/:id` — good place to add a diff tab/section

## Technical Notes

### Diff Viewer Component

Use `react-diff-viewer-continued` for rendering:
- Unified diff mode
- Syntax highlighting via included Prism support
- Dark theme matching dashboard design tokens
- Collapsible file sections

### Component Structure

```
DiffView/
  DiffView.tsx          — Main container, fetches diff data
  DiffView.module.css
  FileList.tsx          — Sidebar listing changed files with status badges
  FileDiff.tsx          — Single file diff with collapsible hunks
  DiffStats.tsx         — Summary bar (X files, +Y, -Z)
```

### Integration Points

1. **AgentCard** — Add "View Diff" button (only shown when agent has a worktree with changes)
2. **AgentDetailPage** — Add diff section below the existing terminal/log view
3. **AgentTerminal** — Add role badge to modal header next to agent ID
4. **New route** — Consider `/agents/:id/diff` or keep as a section within agent detail

### Data Fetching

```typescript
// New TanStack Query hooks
useProjectDiff(slug, branch)   // GET /api/projects/{slug}/diff?branch={branch}
useProjectBranches(slug)       // GET /api/projects/{slug}/branches
```

### Role Badge in Terminal

The AgentTerminal currently receives only `agentId`. It needs the agent's role to display the badge. Options:
- Pass `role` as a prop (requires AgentCard/caller to provide it)
- Fetch agent details inside the terminal (adds a query but is self-contained)
- Preferred: pass as prop since callers already have the agent object
