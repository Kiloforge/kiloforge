# Specification: Standardize Domain Sentinel Errors and Store Returns

**Track ID:** refactor-error-standardization_20260310041003Z
**Type:** Refactor
**Created:** 2026-03-10T04:10:03Z
**Status:** Draft

## Summary

Remove duplicate error types from the service layer, standardize all store methods to return domain sentinel errors, and update callers to use `errors.Is()` instead of type assertions.

## Context

The service layer defines custom `ProjectExistsError` and `ProjectNotFoundError` struct types that duplicate sentinel errors already in `domain/errors.go`. Three stores return non-standard error formats instead of domain sentinel errors. The REST handler uses `errors.As()` type assertions instead of `errors.Is()` with sentinels.

## Codebase Analysis

- **C6:** `service/project_service.go` — `ProjectExistsError`/`ProjectNotFoundError` duplicate `domain.ErrProjectExists`/`domain.ErrProjectNotFound`
- **A11:** `agent_store.go:67-82` — `FindAgent()` doesn't return `domain.ErrAgentNotFound`
- **A12:** `project_store.go:24-38` — `Get()` returns `(zero, false)` instead of sentinel error
- **A13:** `pr_tracking_store.go:23-40` — `LoadPRTracking()` doesn't return `domain.ErrPRTrackingNotFound`

## Acceptance Criteria

- [ ] `ProjectExistsError` and `ProjectNotFoundError` removed from service layer
- [ ] Service uses `fmt.Errorf("...: %w", domain.ErrProjectNotFound)` pattern
- [ ] `agent_store.FindAgent()` returns `domain.ErrAgentNotFound` when not found
- [ ] `project_store.Get()` returns `domain.ErrProjectNotFound` when not found
- [ ] `pr_tracking_store.LoadPRTracking()` returns `domain.ErrPRTrackingNotFound` when not found
- [ ] REST handler uses `errors.Is()` for error checking, not `errors.As()` with custom types
- [ ] All existing tests updated and passing
