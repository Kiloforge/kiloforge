## Summary

Add SSH key selection to the dashboard's "Add Project" form. When a user enters an SSH-based remote URL, show a dropdown of available SSH keys from the server. Extend the API to support listing SSH keys and passing the selected key when adding a project.

## Context

The CLI `kf add` command already supports interactive SSH key selection (completed in `ssh-key-selection_20260309100000Z`), but the dashboard UI has no equivalent. The `AddProjectForm` component only has remote URL and name fields. Users adding SSH-based repos via the UI have no way to specify which SSH key to use — the backend uses the system default, which may not be the right key.

## Technical Notes

### SSH Key Dropdown Behavior

1. User types remote URL
2. Frontend detects SSH URL pattern (`git@...` or `ssh://...`)
3. Frontend fetches `GET /-/api/ssh-keys`
4. If keys found → show dropdown with auto-select for single key
5. If no keys → hide dropdown, use system default
6. On submit → include `ssh_key` in request body

### API Schema

```yaml

    comment: string     # Key comment if present

# Updated request

  properties:
    remote_url: string (required)
    name: string (optional)
    ssh_key: string (optional)  # Path to SSH private key
```
