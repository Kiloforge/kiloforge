## Summary

Extract substantial business logic from `cli/implement.go` (411 lines) and `cli/skills.go` (275 lines) into proper service layer methods. Also extract git workflow logic from `cli/push.go` and SSH key discovery from `cli/add.go`.

## Context

Product guidelines mandate "thin adapters" — CLI commands should parse args, construct services, and delegate. Two CLI commands contain significant business logic: `implement.go` has track validation, consent flow, worktree acquisition, and completion callbacks; `skills.go` has update checking, installation, and config management. This violates clean architecture.

- **A1 (Critical):** `cli/implement.go` (411 lines) — track validation, consent, worktree acquisition, completion callbacks, board state transitions
- **A2 (High):** `cli/skills.go` (275 lines) — skill update, GitHub release checking, version comparison, installation flow
- **A3 (Medium):** `cli/push.go:93-143` — git fetch/ahead-behind logic
- **A4 (Medium):** `cli/add.go:199-223` — SSH key discovery and selection
