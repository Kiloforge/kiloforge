# Specification: Fix Project Manager Wiring in REST Server

**Created:** 2026-03-09T12:00:00Z

## Summary

Wire `ProjectService` into the REST server's `APIHandler` so that the "Add Project" and "Remove Project" API endpoints actually work. Currently they return "project management not configured" because `ProjectMgr` is nil.

## Context

The `APIHandler` has a `projectMgr` field and the `AddProject`/`RemoveProject` handlers are fully implemented, but `ProjectMgr` is never set in `APIHandlerOpts` when creating the handler in `server.go:193`. All dependencies needed to create a `ProjectService` are already available on the `Server` struct:

- `s.registry` (`*jsonfile.ProjectStore`) — implements `ProjectStoreWriter`
- `s.client` (`*gitea.Client`) — implements `ProjectGiteaClient`
- `s.cfg` — has `DataDir`, `OrchestratorPort`, `GiteaAdminUser`

## Codebase Analysis

### The bug
`backend/internal/adapter/rest/server.go:193` — `APIHandlerOpts` is created without `ProjectMgr`:
```go
apiHandler := NewAPIHandler(APIHandlerOpts{
    Agents:     s.store,
    Quota:      s.quotaReader,
    // ... other fields ...
    // ProjectMgr is MISSING
})
```

### The fix
Create a `ProjectService` and pass it:
```go
projectSvc := service.NewProjectService(s.registry, s.client, service.ProjectServiceConfig{
    DataDir:          s.cfg.DataDir,
    OrchestratorPort: s.cfg.OrchestratorPort,
    GiteaAdminUser:   s.cfg.GiteaAdminUser,
})

apiHandler := NewAPIHandler(APIHandlerOpts{
    // ... existing fields ...
    ProjectMgr: projectSvc,
})
```

### Files to modify
- `backend/internal/adapter/rest/server.go` — add `ProjectMgr: projectSvc` to `APIHandlerOpts`
