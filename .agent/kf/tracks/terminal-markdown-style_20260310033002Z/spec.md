# Specification: Agent Terminal Markdown Rendering and Terminal-Style Design

**Track ID:** terminal-markdown-style_20260310033002Z
**Type:** Feature
**Created:** 2026-03-10T03:30:02Z

## Summary

Render markdown in agent chat messages (code blocks, bold, lists, inline code, links) and restyle the terminal UI with a terminal-inspired aesthetic — monospace font, dark background, block-style text — while preserving the existing badge and status chip design.

## Context

Currently the agent terminal displays raw text with no formatting. When Claude outputs markdown (code blocks, lists, bold text), it appears as literal markdown syntax. The terminal also uses the standard app styling rather than a terminal-inspired look. Users familiar with CLI tools would find a terminal aesthetic more natural and easier to read.

## Codebase Analysis

- **AgentTerminal** (`components/AgentTerminal.tsx`): Modal overlay with header, messages area, input. Messages rendered via `MessageDispatch`
- **MessageDispatch** (`components/terminal/index.ts` or similar): Type-switches on message type (text, tool_use, thinking, turn_start, turn_end, system, error) and renders the appropriate component
- **TextMessage rendering**: Currently renders `msg.text` as plain text in a `<div>` or `<span>`
- **AgentDetailPage TerminalSection**: Inline terminal at bottom of detail page, same message rendering
- **Existing badges**: Turn start/end badges, tool use chips, thinking blocks — these use the status badge color scheme and should be preserved
- **No markdown library** currently in frontend dependencies

## Acceptance Criteria

- [ ] Text messages render markdown: code blocks (with syntax highlighting optional), inline code, bold, italic, lists, links
- [ ] Code blocks use monospace font with copy button
- [ ] Terminal messages area has dark/terminal-style background (dark surface)
- [ ] Message text uses monospace or semi-monospace font (block-style lettering)
- [ ] Existing badges (turn_start, turn_end, tool_use, thinking, status) retain their current colored chip design
- [ ] Input area matches terminal aesthetic
- [ ] Both AgentTerminal (modal) and AgentDetailPage terminal section use the same styling
- [ ] Performance is acceptable — no visible lag rendering large markdown blocks
- [ ] Links in markdown are clickable and open in new tab

## Dependencies

None

## Out of Scope

- Syntax highlighting for code blocks (nice-to-have, not required)
- LaTeX/math rendering
- Image rendering in messages
- Backend changes

## Technical Notes

### Markdown Library
Use `react-markdown` (lightweight, widely used) or `marked` + `DOMPurify` for sanitized HTML rendering. `react-markdown` is preferred as it integrates naturally with React components.

```
npm install react-markdown
```

### Terminal Styling Approach
- Dark background (`#1a1a2e` or similar dark surface from CSS variables)
- Monospace font (`"JetBrains Mono", "Fira Code", "Cascadia Code", monospace`)
- Slightly reduced font size (13-14px)
- Green/light accent for text on dark background
- Keep existing badge components unchanged — they already have good color-coded styling
- Subtle left border or background tint to differentiate message types (text vs tool_use vs thinking)

### Component Structure
- Wrap text message content in `<ReactMarkdown>` instead of raw text
- Add custom renderers for code blocks (pre/code with monospace, dark bg, padding)
- Terminal container gets new CSS class with dark background, monospace font
