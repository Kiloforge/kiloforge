# Specification: Navigation Bar Design Polish and Agent Status Popover

## Summary

Improve the navigation bar's visual design — fix link spacing, add an "Agents" label to the status badges section, and add a clickable popover on agent status chips that shows a summary of agents in that status with quick links to agent details.

## Context

The current nav bar is functional but has visual issues: links in the right-side `<nav>` have no gap between them, the overall design is plain, and the agent status chips (AgentHistogram) are not interactive. Users should be able to click a status badge to see which agents are in that state without navigating away.

## Codebase Analysis

- **Nav bar** is inline in `App.tsx` (lines 110-124) — `<header>` with flexbox `space-between`
- **Left side** (`headerLeft`): "kiloforge" title, `ConnectionStatus` badge, `AgentHistogram` chips. Uses `gap: 12px`
- **Right side** (`<nav>`): "Agents" Link + conditional "Gitea" link. **No gap defined** — links sit flush together
- **AgentHistogram** (`components/AgentHistogram.tsx`): Filters active agents, renders flat chips with count + status. Not clickable. Returns `null` when no active agents
- **CSS** (`App.module.css`): `.link` has `font-size: 14px`, accent color, underline on hover. No padding or margin between links
- **Agent data** available via `useAgents()` hook in App.tsx — full agent list with status, name, role, ref

## Acceptance Criteria

- [ ] Nav links have proper spacing (gap between them)
- [ ] Nav links are visually improved (pill/button style or better typography)
- [ ] Agent status section has a clear "Agents" label or visual separator
- [ ] Clicking an agent status chip opens a popover/dropdown showing agents in that status
- [ ] Popover shows agent name, role, ref for each agent in that status
- [ ] Each agent in the popover links to its detail page (`/agents/{id}`)
- [ ] Popover closes on outside click or Escape key
- [ ] Design is cohesive with existing color scheme and badge styles

## Dependencies

None

## Out of Scope

- Backend changes
- Agent lifecycle actions in popover (stop/resume — covered by agent-lifecycle tracks)
- Mobile responsive nav

## Technical Notes

- Add `gap` to the `<nav>` element or use styled link components with padding
- Popover can be a simple absolutely-positioned div with click-outside detection
- Group agents by status in the popover, or show flat list filtered by clicked status
- Consider showing completed/failed agents count too (currently histogram only shows running/waiting)
