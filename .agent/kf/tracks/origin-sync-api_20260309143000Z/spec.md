## Summary

Add REST API endpoints for pushing local main to a configurable remote branch and pulling latest state from upstream. The push targets a user-specified remote branch (not necessarily `main`), allowing non-destructive contribution that preserves the upstream merge workflow.

## Context

`kf push` exists as a CLI command but has no REST API equivalent, so the dashboard can't trigger it. There's also no pull/fetch capability at all. Users need:

1. **Push local main → remote branch** — push to a branch like `kf/main` or `kf/<slug>` on the upstream remote, so they don't clobber the remote's main branch. This lets users (or other developers) perform a proper merge/PR process on the upstream repo.
2. **Pull latest from upstream** — fetch and merge upstream changes into the internal Gitea clone to stay current.

The existing `cli/push.go` already supports `--branch` flag and SSH key env setup. The API needs to expose this functionality with the remote branch as a required parameter.

## Technical Notes

### Push endpoint
```
POST /api/projects/{slug}/push
Body: { "remote_branch": "kf/main" }
Response: { "success": true, "local_branch": "main", "remote_branch": "kf/main", "commits_pushed": 3 }
```

The push always pushes local `main` to `origin/<remote_branch>`. This means:
- `git push origin main:refs/heads/kf/main` — push local main as remote `kf/main`
- Force push is NOT allowed by default (add `"force": true` option for advanced use)

### Pull endpoint
```
POST /api/projects/{slug}/pull
Body: { "remote_branch": "main" }  // optional, defaults to "main"
Response: { "success": true, "commits_pulled": 5, "new_head": "abc123" }
```

Pull fetches from origin and attempts fast-forward merge. If diverged, returns error with ahead/behind counts rather than forcing.

### Sync status endpoint
```
GET /api/projects/{slug}/sync-status
Response: {
  "local_branch": "main",
  "remote_url": "git@github.com:user/repo.git",
  "ahead": 3,
  "behind": 0,
  "last_fetch": "2026-03-09T14:00:00Z",
  "status": "ahead"  // "synced" | "ahead" | "behind" | "diverged" | "unknown"
}
```

### Git operations
Extract a `GitSync` adapter (or extend existing git adapter) that handles:
- `FetchOrigin(projectDir, sshKeyPath)` — git fetch origin
- `PushToRemote(projectDir, localBranch, remoteBranch, sshKeyPath)` — git push origin local:remote
- `PullFromRemote(projectDir, remoteBranch, sshKeyPath)` — git fetch + merge
- `SyncStatus(projectDir)` — git rev-list counts
