## Summary

Fix agent spawning to pass `--dangerously-skip-permissions` to `claude -p` (required for non-interactive mode). Add a one-time user consent flow before the first agent spawn — store consent in SQLite so subsequent spawns proceed without prompting.

## Context

The spawner currently invokes `claude -p "<prompt>" --session-id <id> --output-format stream-json` without `--dangerously-skip-permissions`. In non-interactive (`-p`) mode, Claude Code will hang or fail on tool permission prompts since there's no human to approve them. The design doc (`backend/docs/design-agent-orchestration.md:111`) intended this flag to be passed but it was never wired in.

Since this flag grants agents unrestricted tool access, users should explicitly consent before any agent is spawned. Once consented, store the flag in the database so they aren't asked again.

### Spawner change

    "-p", prompt,

### Consent check flow

1. CLI (`kf implement`): check config store → if not consented, prompt → store → proceed
2. REST API: check config store → if not consented, return 403 with `{"error": "agent_permissions_not_consented", "message": "..."}`
3. Dashboard: on 403, show consent dialog → `POST /api/consent/agent-permissions` → retry spawn
