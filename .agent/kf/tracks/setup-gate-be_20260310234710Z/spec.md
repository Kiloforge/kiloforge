## Summary

Add 428 "kiloforge setup required" check to `RunAdminOperation` endpoint, matching the existing pattern from `GenerateTracks` and `SpawnInteractiveAgent`. Currently admin operations (bulk archive, compact archive, report) can be triggered on projects that haven't completed setup, which will fail confusingly.

## Context

The setup prerequisite system (`setup-prereq-be` track) added 428 checks to `GenerateTracks` and `SpawnInteractiveAgent`, but `RunAdminOperation` was missed. All three endpoints spawn agents that run kf-* skills requiring conductor artifacts — if setup hasn't been done, they fail with cryptic errors instead of a clear "setup required" response.

## Codebase Analysis

### Existing 428 pattern (`api_handler.go`)

- `GenerateTracks` (line ~1362): `if slug := h.checkSetup(projectSlug); slug != "" { return gen.GenerateTracks428JSONResponse{...} }`
- `SpawnInteractiveAgent` (line ~344): Same pattern, conditional on project being specified
- `checkSetup()` helper (line ~250): Returns empty string if setup complete, project slug if not
- `isSetupRequired()` (line ~268): Checks `.agent/conductor/product.md` existence

### Missing check

- `RunAdminOperation` (line ~1436): Has 401 (auth), 403 (consent), 412 (skills) checks but NO 428 (setup) check. Admin operations like bulk-archive, compact-archive, and report all require conductor artifacts.

### OpenAPI schema (`api/openapi.yaml`)

- `RunAdminOperation` response definitions include 401, 403, 412, 429, 500 — missing 428
- `SetupRequiredResponse` schema already exists (lines 1167-1176)

if req.Body.Project != nil && *req.Body.Project != "" {
