## Summary

Comprehensive E2E tests for the project kanban board: displaying tracks as cards in columns, moving cards between columns, syncing board state with backend, and verifying column transitions match agent lifecycle, covering happy path, edge cases, and expected failures.

## Context

Kiloforge's dashboard includes a kanban board that organizes tracks into columns: Backlog, In Progress, In Review, and Done. Each track appears as a card in the column corresponding to its current status. Users can move cards between columns (via drag-and-drop or API), and the board syncs state with the backend. The board also receives real-time updates via SSE (`board_update` events) when tracks change status due to agent activity.

## Codebase Analysis

### Board API endpoints

- `GET /api/projects/{projectId}/board` — returns board state with columns and card positions
- `PUT /api/projects/{projectId}/board/cards/{trackId}` — move a card to a different column
- `POST /api/projects/{projectId}/board/sync` — sync board state from backend track statuses

### Frontend board component

The React kanban board component renders four columns, each containing track cards. Cards display the track title, ID, and a status indicator. Cards can be moved between columns via drag-and-drop (or button controls). The board subscribes to SSE `board_update` events for real-time updates.

### Track domain model

Tracks have a status field that maps to kanban columns:
- `backlog` -> Backlog column
- `in_progress` -> In Progress column
- `in_review` -> In Review column
- `done` / `complete` -> Done column

### SSE integration

The `/events` SSE endpoint emits `board_update` events when a card moves, and `track_update` events when a track's status changes. The board component listens for these to update without polling.

### Card movement verification

After moving a card:
1. Assert the card visually appears in the target column (Playwright locator)
2. Assert the source column no longer contains the card
3. Assert column count badges updated
4. Verify via API (`GET /api/projects/{id}/board`) that the backend state matches

### Drag-and-drop testing with Playwright

Use Playwright's drag-and-drop API:

If drag-and-drop is not implemented, test via button controls or direct API calls with UI verification.
