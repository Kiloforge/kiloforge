## Summary

Update the dashboard's interactive agent terminal to display structured output from the enriched WebSocket protocol — tool calls, thinking blocks, text content, and turn boundaries instead of flat raw text.

## Context

The backend track (`sdk-agent-migration-be_20260310014147Z`) migrates the agent spawner to the Claude Agent SDK and enriches the WebSocket protocol with structured message types: `turn_start`, `text`, `tool_use`, `thinking`, `turn_end`, and `system`. The frontend currently displays all agent output as flat text strings via the `output` message type. This track updates the frontend to render the structured messages with appropriate visual treatment.

## Technical Notes

### New WSMessage Structure

Extend the `WSMessage` type to carry structured data:
```typescript
type WSMessage = {
  type: 'input' | 'text' | 'tool_use' | 'thinking' | 'turn_start' | 'turn_end' | 'system' | 'status' | 'error' | 'output';
  text?: string;
  timestamp: number;
  // Structured fields
  turnId?: string;
  toolName?: string;
  toolId?: string;
  toolInput?: Record<string, unknown>;
  thinking?: string;
  costUsd?: number;
  usage?: { input_tokens: number; output_tokens: number; cache_read_tokens: number; cache_creation_tokens: number };
  subtype?: string;
  data?: Record<string, unknown>;
};
```

### Component Architecture

- `TurnSeparator` — horizontal rule with turn number
- `TextBubble` — existing text rendering (reused)
- `ToolUseBubble` — tool name badge + collapsible JSON input
- `ThinkingBubble` — muted italic block, collapsible
- `TurnEndSummary` — inline cost/token summary after turn separator
- `SystemBubble` — warning/error/info with severity icon

### Backward Compatibility

Legacy `output` messages are treated as `text` messages for rendering. The hook maps `output` → `text` internally so old backends still work.
