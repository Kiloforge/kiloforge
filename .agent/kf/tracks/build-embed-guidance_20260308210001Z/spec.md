## Technical Notes

**Patterns to document:**

1. **Frontend embed pattern:**
   - `//go:embed dist/*` in `embed.go` тАФ no build tags, no `.gitkeep`
   - `ensure-dist` Makefile target creates placeholder `index.html` when dist is empty
   - `dist/` is in `.gitignore` тАФ never committed
   - Real assets come from `make build-frontend` (npm ci + npm run build)

2. **Build entry point:**
   - All builds go through `Makefile` тАФ `make build`, `make dev`, `make test`
   - No direct `go build` or `go test` from the repo root
   - `make build-backend` depends on `ensure-dist` automatically
   - Build output goes to `.build/` (gitignored), never `bin/`

3. **VCS support:**
   - Never use `-buildvcs=false` тАФ Go's built-in VCS stamping is valuable
   - Use `GOFLAGS=-buildvcs=false` env var only in CI where git info is unavailable
   - Local builds always have full VCS metadata
