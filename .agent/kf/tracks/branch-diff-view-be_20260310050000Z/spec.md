## Summary

Add a REST API endpoint that returns structured git diffs between any branch and main, plus an endpoint to look up which agent is working on a given branch. This powers the frontend diff viewer that replaces Gitea's PR review UI.

## Context

The project is moving away from Gitea as the review/diff interface. Instead, the dashboard will provide a built-in diff viewer for any agent's working branch vs main. The backend needs to provide structured diff data (not raw patch text) so the frontend can render a rich file-by-file diff view.

Agents work in worktrees with branches like `worker-1`, `worker-2`, etc., and create implementation branches named after their track ID. The pool already tracks which agent owns which worktree and branch.

## Codebase Analysis

- **Git operations** exist in `internal/adapter/git/sync.go` — currently only sync status (ahead/behind), no diff capability
- **Pool** in `internal/adapter/pool/pool.go` — tracks worktree-to-agent-to-branch mappings via `Worktree.TrackID`, `Worktree.AgentID`, `Worktree.Branch`
- **OpenAPI spec** at `backend/api/openapi.yaml` — all endpoints are schema-first with generated handler stubs
- **REST handler** at `internal/adapter/rest/api_handler.go` — implements generated `ServerInterface`
- **Agent domain** has `WorktreeDir` field linking agents to their worktree path

## Technical Notes

### Diff Service

Create a new git diff adapter (`internal/adapter/git/diff.go`) that:
1. Runs `git diff main...{branch} --stat` for summary stats
2. Runs `git diff main...{branch} --no-color -U3` for unified diff output
3. Parses the unified diff into structured `FileDiff` objects with hunks

### Structured Response

```json
{
  "branch": "track-id-branch",
  "base": "main",
  "stats": {
    "files_changed": 5,
    "insertions": 120,
    "deletions": 30
  },
  "files": [
    {
      "path": "internal/core/service/foo.go",
      "old_path": "",
      "status": "modified",
      "insertions": 20,
      "deletions": 5,
      "is_binary": false,
      "hunks": [
        {
          "old_start": 10,
          "old_lines": 7,
          "new_start": 10,
          "new_lines": 12,
          "header": "@@ -10,7 +10,12 @@ func Foo() {",
          "lines": [
            {"type": "context", "content": " existing line", "old_no": 10, "new_no": 10},
            {"type": "delete", "content": "-removed line", "old_no": 11, "new_no": null},
            {"type": "add", "content": "+added line", "old_no": null, "new_no": 11}
          ]
        }
      ]
    }
  ]
}
```

### Branch Listing

Leverage `pool.Pool` to enumerate active worktrees with their branch, agent ID, and track ID. Expose via a simple endpoint so the frontend can show a branch picker.

### Diff Parsing

Write a standalone diff parser (`internal/adapter/git/diffparse.go`) that converts unified diff text into the structured format. This is pure string parsing with no external dependencies — test it thoroughly with table-driven tests covering: new files, deleted files, renamed files, binary files, empty diffs, files with no newline at EOF.
