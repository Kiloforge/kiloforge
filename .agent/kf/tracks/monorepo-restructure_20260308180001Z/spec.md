## Summary

Restructure the project into a monorepo with `backend/` (Go) and `frontend/` (React/Vite/TypeScript) directories. Configure the build so the Go binary embeds the compiled frontend assets for single-binary deployment. Set up Vite dev proxy so the frontend dev server forwards API calls to the backend.

## Context

The current dashboard is a vanilla JS/CSS/HTML app embedded via `//go:embed static/*` in `internal/adapter/dashboard/embed.go`. To build a full-featured React frontend, we need proper project structure, build tooling, and a dev workflow that supports hot-reload on the frontend while hitting the real backend API.

## Codebase Analysis

- **Current structure:** Go module at root (`go.mod` with `module kiloforge`), all code under `cmd/`, `internal/`, `docs/`
- **Current embed:** `internal/adapter/dashboard/embed.go` embeds `static/*` (3 files: index.html, app.js, style.css)
- **Go module:** `module kiloforge`, Go 1.24.4, deps: cobra, uuid only
- **No existing build pipeline:** Makefile exists but is minimal
- **Import paths:** ~40+ files use `kiloforge/internal/...` imports — all must be updated if module path changes
- **Dashboard routes:** `RegisterRoutes(mux)` mounts `/api/*`, `/events`, `/` on the relay server mux
- **Unified server:** Single port (3001) serves webhooks, dashboard, lock service, and Gitea proxy

### Embed Strategy

Replace the current `static/*` embed with `dist/*`:

```go
//go:embed dist/*
var embeddedFiles embed.FS
```

The `dist/` directory is populated by `cd frontend && npm run build` which outputs to `backend/internal/adapter/dashboard/dist/`.
