## Summary

Comprehensive E2E tests for the distributed tracing UI: listing traces, viewing trace detail with span hierarchy, timeline visualization, and trace filtering, covering happy path, edge cases, and expected failures.

## Context

Kiloforge includes a distributed tracing system that captures spans across agent operations, webhook events, and internal processing. The tracing endpoints are:

- `GET /api/traces` — list traces with IDs, durations, and span counts
- `GET /api/traces/{traceId}` — trace detail with full span hierarchy

The frontend renders a trace list page and a trace detail page with a timeline visualization showing parent-child span relationships. Real-time updates arrive via `trace_update` SSE events.

These E2E tests verify the full stack — API responses, span hierarchy rendering, timeline visualization, and SSE-driven updates — using seeded trace data and the mock agent infrastructure.

## Codebase Analysis

### Relevant endpoints

- Trace list: `GET /api/traces` — returns array of traces with `trace_id`, `duration_ms`, `span_count`, `root_span_name`, `started_at`
- Trace detail: `GET /api/traces/{traceId}` — returns trace with nested spans, each having `span_id`, `parent_span_id`, `name`, `start_time`, `end_time`, `attributes`

### SSE events

- `trace_update` — emitted when new spans are recorded, includes trace ID

### UI components

- Trace list page: table/list of traces with clickable rows
- Trace detail page: span hierarchy tree and timeline visualization
- Span attributes display: project, track, agent, webhook event metadata

## Technical Notes

### Test data seeding

Seed traces with known span hierarchies before each test. Create traces with varying depth (1 span, 3 spans in a chain, 5+ level deep nesting) and varying span counts. Include spans with all relevant attribute types (project slug, track ID, agent ID, webhook event type).

### Timeline verification

Playwright assertions should verify:
- Spans are rendered as horizontal bars
- Parent spans visually contain child spans
- Relative start times are reflected in horizontal positioning
- Duration is reflected in bar width

Use `data-testid` attributes or accessible roles to locate timeline elements.

### Deep nesting test

Create a trace with 6+ levels of nested spans. Verify the UI renders all levels without truncation or layout overflow. Check that indentation or tree lines are visible.

When building this track, use the Playwright MCP skill to verify E2E tests work in the browser. Run tests in headed mode during development for visual verification. Seed trace data via direct service calls or REST API before each test.
