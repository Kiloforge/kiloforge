## Summary

Fix the guided tour — clicking any button returns 400 `{"error":"invalid action, must be accept|advance|dismiss|complete"}`. The frontend sends `{ status: "active", current_step: 0 }` but the backend expects `{ action: "accept", step?: number }`.

## Context

The `PUT /api/tour` endpoint and the `useTour` hook were built by different tracks (BE and FE) with mismatched request body contracts.

## Codebase Analysis

### Backend (`tour_handler.go`)

Expects:
```json
{ "action": "accept", "step": 0 }
```

Valid actions: `accept`, `advance`, `dismiss`, `complete`.

### Frontend (`useTour.ts`)

Sends:
```json
{ "status": "active", "current_step": 0 }
```

The mutation sends a `Partial<TourState>` object directly instead of mapping to the action-based API.

### Fix

Update `useTour.ts` mutation functions to send the correct action-based body:

- `startTour()` → `{ action: "accept" }`
- `advanceStep(step)` → `{ action: "advance", step: step }`
- `dismissTour()` → `{ action: "dismiss" }`
- `completeTour()` → `{ action: "complete" }`
