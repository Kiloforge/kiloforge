## Summary

Add a dashboard admin panel with buttons for conductor management operations (bulk archive, compact archive, report generation). Each operation spawns an interactive agent with the appropriate `kf-*` skill, showing real-time output in an embedded terminal.

## Context

Management operations like bulk-archive, compact-archive, and report generation currently require a human to manually invoke skills via Claude Code CLI. The dashboard already has the interactive agent infrastructure (WebSocket, terminal component) used by track generation. Admin operations should follow the same pattern — spawn an agent, show output, auto-refresh board state on completion.

## Technical Notes

### Backend: New REST endpoint

Add `POST /api/admin/run` endpoint:

```json
{
  "operation": "bulk-archive" | "compact-archive" | "report",
  "project": "optional-project-slug"
}
```

Response: `{ "agent_id": "...", "ws_url": "/ws/agent/..." }`

The handler follows the same pattern as `GenerateTracks`:
1. Map operation to skill prompt (e.g., `"bulk-archive"` → `/kf-bulk-archive`)
2. Spawn interactive agent with the skill prompt
3. Register WebSocket bridge
4. On completion, trigger board sync if operation was archive-related
5. Return agent ID and WS URL

### Frontend: Admin panel component

New `AdminPanel` component with:
- Row of operation buttons (Bulk Archive, Compact Archive, Report)
- Each button triggers `POST /api/admin/run`, then opens agent terminal
- Disabled state while operation is running
- Success/failure indication after completion

Place in `ProjectPage.tsx` as a collapsible "Admin Operations" section below the board.
